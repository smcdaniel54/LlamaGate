# Plugin Templates

Choose the right template for your needs:

## Simple Plugin Template (`simple_plugin.go`) ⭐ Recommended

**Best for:** Beginners, simple plugins, quick prototypes

**Features:**
- ✅ Minimal code (~50 lines)
- ✅ Only 3 required methods
- ✅ Easy to understand
- ✅ Perfect for learning

**Use when:**
- You're new to plugins
- You need a simple plugin quickly
- You want minimal boilerplate

## Full Plugin Template (`plugin_template.go`)

**Best for:** Production plugins, complex workflows, full documentation

**Features:**
- ✅ Complete structure
- ✅ JSON Schema definitions
- ✅ Comprehensive validation
- ✅ Full documentation

**Use when:**
- You need full API documentation
- You want comprehensive validation
- You're building production plugins

## Which Should I Use?

### Start with Simple Template if:
- ✅ You're learning
- ✅ You want something quick
- ✅ Your plugin is straightforward
- ✅ You don't need schemas yet

### Use Full Template if:
- ✅ You need API documentation
- ✅ You want comprehensive validation
- ✅ You're building for production
- ✅ You need full feature set

## Quick Comparison

| Feature | Simple | Full |
|---------|--------|------|
| Lines of Code | ~50 | ~150 |
| Methods | 3 | 3 |
| JSON Schema | Optional | Included |
| Validation | Basic | Comprehensive |
| Time to Create | 5 min | 15 min |
| Best For | Learning | Production |

## Migration Path

You can always start simple and add features later:

1. **Start**: Use `simple_plugin.go`
2. **Add**: JSON Schemas when needed
3. **Enhance**: Add workflows for complex logic
4. **Extend**: Add custom endpoints if needed

The simple template is a perfect starting point!
